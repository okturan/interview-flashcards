# Security policy

## Supported version

The current deployment and the default branch receive best-effort security fixes. Older commits and user-built deployments are not supported versions.

## Reporting a vulnerability

Use GitHub's **Report a vulnerability** form in the repository Security tab. Please include:

- the affected route or component;
- clear reproduction steps;
- the impact you observed; and
- a minimal proof of concept when one is safe to share.

Do not include secrets, personal study data, or destructive payloads. Please allow reasonable time for investigation before public disclosure.

This project is a local-first static application. It has no project-operated account system or application database, but reports involving dependency integrity, the deployed PWA, import/export behavior, or unintended exposure of browser-stored data are in scope.
